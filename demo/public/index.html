Impact

Why it matters

Velocity from AI-assisted development should not outpace assurance. VibeScan targets classes of vulnerabilities in source with severities, precise locations, remediation, safe patterns, and references, so teams fix with intent, not guesswork.

  • Run locally or in CI with pinned, reproducible versions.
  • Governance exports for Cursor, Amazon Q, and policy JSON aligned to your repo.
  • Optional deterministic proof tests for supported finding classes: no live API, no secrets.

Architecture

End-to-end static analysis

Unified parsing and semantic context, rule-based and taint analysis, cross-file route and API reasoning, then reporting and optional CI validation, matching the same mental model as the shipped scanner pipeline.

Runtime scan flow

From CLI to findings

The CLI loads configuration, discovers files, runs scanProjectAsync, applies cross-file audits (middleware, webhooks, OpenAPI drift, third-party surface), then formats output as JSON, SARIF, HTML, or human readable, with optional proof test emission.

Capabilities

One install, many surfaces

CLI & CI

vibescan scan with JSON or SARIF for gates. GitHub Actions template ships with the package.

Reports

Static HTML report, SARIF for toolchains, rich project JSON with evidence fields when enabled.

IDE assist

Generated prompts and rules: paste into assistants; no remote LLM required from VibeScan.

secure-arch

YAML policy with optional code evidence: architecture facts alongside code signals.

Proof tests

Where supported, deterministic node:test harnesses with no base URL and no secrets.

ESLint plugin

Same rule catalog in-editor; prefer CLI for full TypeScript semantic parity on complex repos.
