Catch risky patterns before they ship.

VibeScan is a developer-first security CLI: static JS/TS scanning, optional semantic TypeScript analysis, HTML and SARIF/JSON output, IDE-friendly AI rule export, and architecture checks — no VibeScan API keys. Free (MIT).

Impact

Why it matters

Teams ship faster with AI-assisted coding — but velocity can outpace review. Crypto mistakes, injection sinks, and fragile auth patterns still reach production if nothing catches them in the repo. VibeScan focuses on classes of issues in your source: clear severities, file/line locations, remediation hints, safe examples, and references — so fixes are actionable, not noisy guesswork.

  • Run locally or in CI; pin versions for reproducible builds.
  • Export governance artifacts for Cursor / Amazon Q / markdown policy digests.
  • Optional proof-oriented test generation for supported finding classes (deterministic, no live API).

How it works

From your repo to a report

Point VibeScan at your project. It parses JavaScript and TypeScript (with optional tsconfig-backed semantics), runs rule and taint analyses, understands routes and app structure, and emits human and machine readable output — plus optional secure-arch validation against YAML policy.

flowchart LR R[Repo] --> P[Parse JS/TS] P --> Q[Rules + taint] Q --> A[Route + app audits] A --> F[Findings] F --> O[HTML / SARIF / JSON] O -.-> X[Proof tests] O -.-> Y[export-ai-rules] O -.-> Z[secure-arch]

Under the hood

Scan pipeline (simplified)

The engine walks parsed ASTs, runs pattern rules and taint tracking, builds route intelligence (including Next.js-style handlers when applicable), runs middleware and app-level audits, checks OpenAPI drift when specs are present, analyzes third-party touchpoints, and can attach proof-harness metadata for supported rules.

flowchart TD PF[parseFile] --> RE[runRuleEngine] PF --> TE[runTaintEngine] RE --> RG[Route graph + Next] TE --> RG RG --> MA[Middleware + app audits] MA --> OD[OpenAPI drift] OD --> TP[Third-party surface] TP --> GT[generateTests]

What you get

One install, many surfaces

CLI & CI

vibescan scan with JSON or SARIF for gates and dashboards. GitHub Actions template included in the package.

Reports

Static HTML report, SARIF for SARIF-aware tools, rich project JSON with evidence fields when enabled.

IDE assist

Generated prompts and rules for assistants — paste-in markdown, no remote LLM required from VibeScan.

secure-arch

YAML policy + optional code evidence: validate architecture facts alongside code signals.

Proof tests

Where supported, deterministic node:test files to exercise findings — no base URL, no secrets.

ESLint plugin

Same rule catalog in-editor; use CLI first for full TypeScript semantic parity on complex projects.
