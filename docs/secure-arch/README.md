# Universal secure-architecture rules (`secure-arch`)

This repo includes **`secure-arch`**, a portable CLI and rule pack designed for **any AI-assisted coding tool** (Cursor, Amazon Q, etc.). AI tools should only **maintain YAML settings** under `architecture/secure-rules/`; **static analysis** enforces secure architecture.

## Concepts

| Piece | Role |
|--------|------|
| Settings YAML | Facts about exposure, auth, secrets, CORS, networks, trust boundaries (no “is it secure?” judgments). |
| `@secure-arch/core` | Schema validation, merged facts, architecture rules, optional code evidence. |
| `@secure-arch/cli` | `install`, `init`, `check` commands; `.gitignore` updates. |
| `@secure-arch/adapters` | Generates **instruction files** per vendor (not Cursor-specific logic). |

## Commands (from repo root)

```bash
npm install
npm run build
npx secure-arch install --root .
npx secure-arch init --tool cursor
npx secure-arch init --tool amazonq
npx secure-arch check --root . --code-evidence js-ts --format human
```

Via the published scanner package, the same commands are available under `vibescan secure-arch ...` after installing `@jobersteadt/vibescan`.

- Exit code **1** if any finding is **critical** or **error** (including code-evidence rules such as `ARCH-E001`).
- `--code-evidence off | js-ts | all` — `js-ts` reuses **VibeScan** (`vibescan`); `all` adds Python/Java heuristics.

## CI

```yaml
- run: npm ci
- run: npm run build
- run: npx secure-arch check --root . --code-evidence js-ts --format json
```

## Publishing note

In this monorepo, `@secure-arch/core` depends on `@jobersteadt/vibescan` via a local file reference (`file:../..`). For external npm publish of secure-arch packages, switch that dependency to a **semver range** on the published `@jobersteadt/vibescan` package.
