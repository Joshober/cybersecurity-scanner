VibeScan: Multi-Dimensional Benchmark Analysis for JavaScript Security Scanning

Bridging vulnerability detection and validation in web security through proof-based analysis and comprehensive evaluation across recall, runtime, and signal efficiency

AUTHORS: JOSH OBERSTEADT, ANDREW YOUSSEF

Mentors: Kevin Brunner | Aquilla Galusha

Abstract

Security vulnerabilities in web applications remain a persistent challenge. Veracode’s industry study of applications developed in EMEA reported that roughly 80% contained at least one security flaw,[2] underscoring scale; most static analysis tools still cannot determine whether a vulnerability is genuinely reachable or exploitable. The result is that developers are left managing high volumes of alerts with little confidence about where real risk exists.

We introduce VibeScan, a static analysis framework evaluated on the DVNA benchmark against five widely used peer scanners.[1,5] VibeScan achieved 100% recall across all confirmed vulnerability cases while completing full analysis in under one second, demonstrating that comprehensive detection is achievable within practical CI/CD constraints.

Beyond recall, we introduce a cases per alert metric to evaluate signal efficiency across tools. Our findings show that high detection coverage and low alert noise are not mutually exclusive. This work contributes a proof oriented analysis approach, a multi-dimensional benchmarking methodology, and a reproducible evaluation framework for the JavaScript security tooling community.

Research Gap & Contributions

Why existing tools fall short

  • Static analysis tools do not determine whether vulnerabilities are actually reachable or exploitable
  • Developers are left manually triaging large volumes of low confidence alerts
  • Most scanners stop at syntax or single file data flow
  • No cross file context and no route level reasoning
  • OWASP Top 10 families remain incompletely covered across peer tools

What VibeScan contributes

  • Unified pipeline combining rule detection, taint flow, and route reasoning
  • Auto generated proof test linked to every finding
  • Executable evidence, not just a line number
  • CI recall gates that prevent coverage regression before each merge
  • Addresses the full OWASP Top 10: 2025 risk set

Figure 3 — Recall vs. Raw Alert Volume
Recall vs. raw alert volume on DVNA. X-axis shows issue count on a logarithmic scale (tick labels are actual counts), so both low-volume and high-volume tools are readable on one chart. Point colors match the tool key. Pearson r shown for linear counts.

Tools that generate more alerts do not find more vulnerabilities. ESLint fires over 500 alerts and catches one case out of eleven. VibeScan fires 25 alerts and catches all 11. The negative correlation confirms that alert volume alone is not a reliable measure of scanner quality (see [1]).

Figure 4 — Coverage by Difficulty Band (DVNA Cases)
DVNA adjudicated test cases grouped into difficulty bands using six-tool consensus k (mapped to research Δ = 18.3 − 1.8×k). Bar height is the share of cases in that band each tool detects. Under each band, we list Δ, k/6, n, and which DVNA cases fall in that bucket.

VibeScan maintains full coverage across all difficulty levels, including the most complex DVNA cases in the rightmost bands, which require deeper contextual reasoning across data flow, application behavior, and cross file interactions. In contrast, peer tools perform well on simpler scenarios but show a clear drop in coverage as difficulty increases, highlighting a gap where detection performance declines as analysis becomes more complex.

Figure 5 — Scanner Efficiency (Recall vs. Runtime)
Higher and farther right indicates a better balance of recall and runtime.

VibeScan is the only tool that lands in the high recall and fast runtime region of the chart, finishing full analysis in under one second. CodeQL is both the slowest tool and the lowest performer on recall.
