VibeScan: Security Evaluation of AI-Generated Node.js Applications

An Academic Poster on Experiment Design, Artifact Creation, and Motivation

Cybersecurity Scanner Research Team · 2026

Abstract

AI-assisted development accelerates delivery but can introduce insecure defaults and exploitable logic. We conducted a structured evaluation of VibeScan, a static-analysis security scanner for generated Node.js code.

Our experiment measures detection behavior on intentionally vulnerable benchmark applications and compares VibeScan's findings to baseline tools.

Why We Created It

  • Prompt-based coding often omits threat modeling and secure-by-default design.
  • Generated applications repeatedly show weak secrets, missing auth checks, and injection paths.
  • Existing checks are fragmented, requiring multiple tools and manual interpretation.
  • We needed a reproducible, developer-friendly scanner focused on LLM-generated backend code.

What We Created

VibeScan is a modular static-analysis package that:

  • Builds AST-based route and flow awareness for Express/Node projects.
  • Detects injection, auth, secret, and unsafe dependency patterns.
  • Produces actionable findings and machine-readable output for CI pipelines.
  • Supports benchmark-driven validation and comparative reporting.

Experiment Design

Step 1 — Corpus SelectionUse benchmark applications with known vulnerable patterns.
Step 2 — Ground Truth SetupMap expected vulnerability classes and adjudicate findings.
Step 3 — Tool ExecutionRun VibeScan and baseline scanners under consistent conditions.
Step 4 — AnalysisCompare true positives, coverage breadth, and evidence quality.

Methods

  • Static analysis pipeline: parser + rule engine + sink/source reasoning.
  • Evaluation lens: class-level detection on first-party application code.
  • Output: human-readable reports and structured artifacts for reproducibility.
  • Quality control: manual adjudication to reduce false-positive inflation.

Conclusions and Future Work

The experiment indicates that targeted static analysis can substantially improve vulnerability visibility in AI-generated backend applications. VibeScan demonstrates stronger benchmark detection than lightweight baseline checks while preserving practical integration for developer workflows.

Future work: expand benchmark diversity, include precision/recall reporting across more tools, and evaluate remediation guidance quality in real development pipelines.
