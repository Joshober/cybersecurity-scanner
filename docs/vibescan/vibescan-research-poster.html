VibeScan: Multi-Dimensional Benchmark Analysis for JavaScript Security Scanning

A reproducible multi-dimensional study on adjudicated DVNA with aligned peer tool runs: recall, cases-per-alert precision proxy, per-family coverage, and wall-clock scan time. CCSC 2026.

AUTHORS: JOSH OBERSTEADT, ANDREW YOUSSEF

Mentors: Kevin Brunner | Aquilla Galusha

Abstract

Modern web applications face persistent attacker pressure, and industry datasets report that roughly 80% of applications contain at least one security flaw [2]. Existing static analysis tools detect potential vulnerabilities, but often stop short of contextual reasoning and actionable validation [6]. We introduce VibeScan, a hybrid static + proof-oriented security framework that combines semantic analysis with validation artifacts to help developers confirm exploitability and prioritize remediation. In a reproducible benchmark on DVNA [1] against peer scanners [5], VibeScan achieves the highest core recall (100.0%) while maintaining fast execution (0.86s). Our results show improved actionability by separating detection quality from alert noise and time-to-validate, using recall, cases-per-alert proxy, and runtime as complementary metrics.

Motivation & Threat Landscape

  • 80%+ of applications contain at least one known flaw (industry prevalence). [2]
  • Vulnerability exploitation remains a major breach entry path; web attacks are measured at internet scale. [3,4]
  • False positives and weak presentation reduce static-analysis adoption in practice. [6]

Current tools prioritize detection over validation, leaving developers uncertain about exploitability and remediation priority. This motivates a framework that converts findings into reproducible, evidence-backed decisions.

VibeScan Framework

VibeScan combines three ideas: (1) static + dynamic hybrid analysis, (2) proof-based validation artifacts, and (3) context-aware reasoning across routes, taint propagation, and dependency behavior. Unlike traditional scanners, VibeScan links findings to executable validation evidence, bridging detection and developer trust in remediation workflows.

Figure 3 — Recall vs. Raw Alert Volume
Recall vs. raw issue count on DVNA (log-scaled X). Pearson r shown for linear counts; point colors match the tool key. PNG/PDF exports remain in chart-assets/ optional.

A tool's raw alert count is a poor proxy for scanner quality. eslint-plugin-security emits over 500 alerts yet detects only 1 of 11 benchmark cases, while VibeScan reports 25 alerts and detects all 11. The negative correlation (Pearson r = -0.73) confirms that alert volume and actual detection are largely independent — evaluating scanners by volume alone is misleading.

Five-tool ranking (same composition as the original chart); x-axis recall 20–100%. DVNA 11-case recall matches the matrix for these tools.

This ranked dot plot makes ordering immediate: VibeScan leads with full recall, followed by Bearer and Snyk Code at mid-tier coverage, while CodeQL and Semgrep trail on missed cases. The compact format preserves exact values while reducing clutter versus a second scatter chart.

Figure 5 — Scanner Efficiency (Recall vs. Runtime)
Higher and farther right indicates a better balance of recall and runtime.

VibeScan occupies the strongest overall region by combining the highest recall with fast execution. Bearer and Snyk Code provide moderate recall at slower runtimes, while CodeQL is fast but misses more benchmark cases. This shows that runtime alone is not a reliable indicator of scanner quality.
