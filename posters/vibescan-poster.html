VibeScan
Static security scanner for JavaScript/TypeScript — injection + crypto failures, CI-friendly JSON/SARIF.

What it detects

OWASP A02:2021 crypto failures
OWASP A03:2021 injection issues

Heuristic static analysis: it extracts a route graph (Express-style) and applies rule packs across middleware and call chains.

Crypto

  • deprecated ciphers
  • weak hashing / IV issues
  • hardcoded secrets + JWT weaknesses

Injection

  • SQL / NoSQL injection
  • command / code injection
  • XSS, path traversal, SSRF signals
Exit code is non-zero when it finds severity=critical|error.

CI-friendly outputs

  • --format json for machine parsing
  • --format sarif for GitHub code scanning
  • --export-routes, --export-adjudication for sidecars
# Example (SARIF):
npx @jobersteadt/vibescan scan . --exclude-vendor --format sarif > vibescan.sarif

Quick start

# Install:
npm i @jobersteadt/vibescan

# Scan:
npx vibescan scan . --exclude-vendor

# Add registry checks + npm audit merge:
npx vibescan scan . --exclude-vendor --check-registry --npm-audit --format json > vibescan.json
For GitHub Packages installs, configure npm scope to npm.pkg.github.com.

Turn features on

  • --rules crypto,injection
  • --no-crypto or --no-injection
  • --check-registry (SLOP-001 signal)
  • --skip-registry (offline)
  • --generate-tests scaffolds Jest stubs
  • --manifest, --build-id for run metadata

Outputs you can export

  • vibescan.sarif (GitHub code scanning)
  • vibescan.json (structured findings)
  • routes.json (static route inventory)
  • adjudication.json + .csv
Route extraction is static: the tool does not execute prompts or call your model.
