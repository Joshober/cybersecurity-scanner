VibeScan: End-to-End Static Analysis with
Cross-File Reasoning and Proof Validation

Deeper JavaScript security analysis that goes beyond pattern matching — combining taint tracking, cross-file reasoning, route/API analysis, and executable proof generation to achieve measurable vulnerability coverage on the DVNA benchmark.

Abstract

VibeScan is a JavaScript security scanner designed to go beyond shallow pattern matching by combining rule-based detection, taint analysis, cross-file reasoning, route and API analysis, and executable proof validation. In this work, we evaluate VibeScan against several existing tools using the DVNA benchmark and an expanded rule-family corpus. VibeScan achieved 11/11 recall on DVNA, outperforming all evaluated tools, and was the only system in the comparison to reach full analysis depth. These results suggest that deeper static analysis paired with proof-oriented validation can improve both vulnerability coverage and confidence in reported findings.

Key Points

  • 11/11 recall on the DVNA benchmark — no other evaluated tool matched this.
  • Only tool to reach full analysis depth, extending through route/API reasoning and proof generation.
  • Cross-file reasoning and route/API analysis improved coverage on harder, low-consensus cases.
  • Proof generation and CI recall gates turned findings into measurable validation artifacts.
  • Stronger recall without excessive noise — 25 alerts vs. up to 493 from pattern-only tools.

Benchmark

The Damn Vulnerable NodeJS Application (DVNA) provides 11 ground-truth vulnerability cases spanning injection, redirect, deserialization, cryptography, access control, and XSS families. Six tools were evaluated on first-party DVNA source using line-anchored case matching.

Conclusion

VibeScan showed that deeper JavaScript security analysis can improve benchmark recall while still supporting practical validation workflows. Cross-file reasoning, route/API analysis, and executable proof generation helped move the tool beyond simple detection toward measurable security assurance. These results suggest that combining semantic depth with proof-oriented validation is a viable path toward higher-confidence static analysis.

Limitations

  • Benchmark results are based primarily on DVNA and an expanded corpus; broader benchmarks may yield different rankings.
  • The precision proxy (cases per alert) is not the same as a fully adjudicated precision study with human-labeled FP/TP.
  • Historical peer runtime data was limited in the available committed artifacts; timings are environment-dependent.
